# Zephyr Blog frontend style guide

Zephyr Blog 是一个带东方纸感和开发者气质的个人博客。它不追求通用 SaaS 感，也不做纯技术站的冷硬黑白。设计应该像一本放在木桌上的技术札记：温润、安静、有一点诗意，但仍然清晰、克制、可阅读。

## 1. 设计关键词

- 东方纸感
- 暖棕与陶土色
- 玻璃拟态
- 昼夜切换
- 轻量动效
- 技术札记
- 中文 serif 氛围
- 安静、克制、留白

不要把页面做成：

- 冷白企业后台
- 赛博朋克 neon dashboard
- 过度卡通的个人主页
- 素材堆叠型 landing page
- 大面积高饱和渐变站点

## 2. 技术基线

当前项目使用：

```text
Astro 6
Tailwind CSS 4
MDX
React islands
```

主要样式入口：

```text
src/styles/global.css
```

全局背景组件：

```text
src/components/GlobalBackground.astro
```

导航组件：

```text
src/components/Header.astro
```

博客文章布局：

```text
src/layouts/BlogPost.astro
```

新增页面和组件时，优先复用现有 CSS 变量、字体类和 `.glass-panel`。不要另起一套颜色系统。

## 3. 色彩系统

颜色通过 CSS variables 管理，浅色和深色模式共用语义 token。

### 浅色模式

```css
--color-brand: #a44525;
--color-primary: #3e2723;
--color-secondary: #5d4037;
--color-tertiary: #8d6e63;
--color-bg-base: #fdfbf7;
--color-border: #d7ccc8;
--color-glass-bg: rgba(255, 255, 255, 0.5);
--color-glass-border: rgba(255, 255, 255, 0.6);
--color-glass-shadow: rgba(0, 0, 0, 0.05);
--color-bg-gradient-from: #ebe3d5;
--color-bg-gradient-via: #eef2ea;
--color-bg-gradient-to: #ffffff;
```

视觉感受：宣纸底、暖棕文字、陶土色强调、低对比边框。

### 深色模式

```css
--color-brand: #e05e36;
--color-primary: #e8e0d5;
--color-secondary: #a39b93;
--color-tertiary: #6b635c;
--color-bg-base: #0f0d0c;
--color-border: #2a2522;
--color-glass-bg: rgba(20, 18, 17, 0.6);
--color-glass-border: rgba(255, 255, 255, 0.08);
--color-glass-shadow: rgba(0, 0, 0, 0.3);
--color-bg-gradient-from: #1a1612;
--color-bg-gradient-via: #0f1410;
--color-bg-gradient-to: #161b17;
```

视觉感受：夜色、微暖文字、低亮度背景、品牌色稍亮。

### 使用规则

- 主标题用 `var(--color-primary)`。
- 正文和说明用 `var(--color-secondary)`。
- 弱信息、编号、meta 用 `var(--color-tertiary)`。
- 可点击重点和品牌强调用 `var(--color-brand)`。
- 边框统一用 `var(--color-border)` 或其透明变体。
- 禁止随意写新的蓝色、紫色、绿色作为主色。
- 若需要状态色，保持低饱和，并只在局部使用。

## 4. 字体系统

全局默认：

```css
font-family: ui-sans-serif, system-ui, sans-serif;
```

项目已引入：

```css
Noto Serif SC
Playfair Display
Cormorant Garamond
```

可用工具类：

```css
.font-serif-sc { font-family: 'Noto Serif SC', serif; }
.font-playfair { font-family: 'Playfair Display', serif; }
.font-cormorant { font-family: 'Cormorant Garamond', serif; }
```

### 字体使用

- 英文大标题：`font-playfair`。
- 中文标题、中文诗性描述：`font-serif-sc`。
- 品牌标识、细窄 uppercase：`font-cormorant`。
- 技术编号、路径、版本号：`font-mono`。
- 长正文优先保证阅读性，不要为了风格牺牲可读性。

### 标题建议

```html
<h1 class="text-5xl md:text-7xl font-playfair font-bold tracking-tight text-[var(--color-primary)]">
```

```html
<h2 class="text-3xl md:text-4xl font-playfair font-bold text-[var(--color-primary)]">
```

中文副标题：

```html
<p class="text-sm font-serif-sc tracking-widest text-[var(--color-secondary)]">
```

## 5. 布局原则

### 页面宽度

- 首页主容器：`max-w-6xl mx-auto px-6`。
- 文章阅读宽度：正文建议 `720px` 左右。
- 内容区不要贴边，移动端至少保留 `px-6`。

### 留白

- 首屏允许大留白，营造安静感。
- 卡片内部 padding 建议 `p-4` 到 `p-6`。
- 模块之间使用 `py-10`、`py-16` 或整屏 section。
- 不要把内容堆满屏幕。

### 网格

卡片列表优先使用：

```html
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-6">
```

文章列表可使用：

```html
<div class="grid grid-cols-1 md:grid-cols-2 gap-6">
```

## 6. 组件风格

### 玻璃面板

复用 `.glass-panel`：

```css
.glass-panel {
  background: var(--color-glass-bg);
  backdrop-filter: blur(16px);
  -webkit-backdrop-filter: blur(16px);
  border: 1px solid var(--color-glass-border);
  box-shadow: 0 10px 30px -10px var(--color-glass-shadow);
}
```

适合：

- 标签 chip
- 浮层
- 音乐播放器
- 局部信息卡

不适合：

- 大段正文容器
- 所有卡片无脑套用
- 高密度列表

### 卡片

基础卡片建议：

```html
<div class="bg-[var(--color-bg-base)] rounded-xl border border-[var(--color-border)] p-4 shadow-sm hover:shadow-md transition-shadow duration-500">
```

技术感卡片可加入：

```html
<div class="text-[10px] text-[var(--color-tertiary)] uppercase tracking-wider font-mono">
  <span>/ SKILL_01</span>
  <span>v1.0.0</span>
</div>
```

### 按钮

主按钮：

```html
<a class="rounded-lg bg-[var(--color-brand)] text-white font-bold tracking-widest hover:brightness-110 hover:-translate-y-0.5 transition-all shadow-md shadow-[var(--color-brand)]/20">
```

次级按钮：

```html
<a class="rounded-lg border border-[var(--color-border)] text-[var(--color-primary)] hover:bg-[var(--color-primary)]/5 transition-colors">
```

按钮文案要短，不要写营销腔。

### 导航

当前导航是悬浮透明风格：

```text
fixed top-0
bg-transparent
backdrop-blur
font-serif-sc
tracking-widest
border-b
```

新增导航项时保持：

- 中文短文案
- 下划线 hover 动效
- 不要加厚重背景
- 不要把导航做成后台菜单

## 7. 背景与动效

### 背景

背景由 `GlobalBackground.astro` 管理：

- 浅色：纸感渐变 + 轻噪点 + 太阳线稿。
- 深色：夜色渐变 + 流星 + 月亮线稿。

不要在页面里再叠一套大背景图。需要局部氛围时，使用低透明度装饰线、边框、点状元素。

### 动效

当前已有：

```text
wordIn
marquee
pulse-badge
meteor-fall
flip-card
celestial sun/moon transition
```

动效原则：

- 慢一点，轻一点。
- 进入动画不超过 0.8s。
- hover 可用 `translate-y-0.5`、`scale-105`、`brightness-110`。
- 避免大幅旋转、弹跳、频闪。
- 深色流星只在 dark 模式显示。

### 可访问性

后续可以补 `prefers-reduced-motion`。新增复杂动画时必须考虑禁用方案。

## 8. 内容语气

这个博客不是公司官网，文字可以有人味。

推荐语气：

- 简洁
- 有一点诗性
- 偏个人札记
- 技术表达准确
- 中文自然，不堆成语

示例：

```text
在前端工程化、微服务后端架构以及读书随笔中寻找平衡点。
```

```text
致虚极，守静笃。在代码与现实的交错中探索。
```

避免：

```text
打造极致体验，赋能未来技术生态。
```

```text
这是一个充满创新与无限可能的技术博客平台。
```

## 9. Markdown 与文章页

文章页应该优先保证阅读体验：

- 正文宽度控制在 `720px` 左右。
- 行高保持宽松。
- 标题层级清楚。
- 代码块要和主题色协调。
- 图片圆角建议 `12px`。
- 日期、更新时间使用 `var(--color-secondary)` 或 `var(--color-tertiary)`。

文章不要过度使用玻璃卡片。正文区域应该像纸面，而不是 dashboard panel。

## 10. 暗色模式要求

任何新增组件都必须检查暗色模式：

- 不要写死 `text-black`、`text-white`，除非确实是品牌按钮。
- 不要写死 `bg-white`、`bg-black`。
- 使用 `var(--color-*)` token。
- 边框在暗色模式要足够低调。
- hover 背景用透明色，如 `bg-[var(--color-primary)]/5`。

## 11. Tailwind 写法约定

优先使用项目 token：

```html
text-[var(--color-primary)]
bg-[var(--color-bg-base)]
border-[var(--color-border)]
text-[var(--color-brand)]
```

不要新增散乱色值：

```html
<!-- 不推荐 -->
<div class="text-zinc-900 bg-white border-gray-200">
```

如果必须新增颜色，先确认是否能转成 CSS variable。

## 12. 文件与组件命名

- Astro 组件使用 PascalCase，例如 `ArticleCard.astro`。
- 页面路径保持清晰，例如 `src/pages/projects/index.astro`。
- 全局样式只放基础 token、动画、全局工具类。
- 页面专属交互可以写在页面局部 `<style>`，但不要复制一大套 token。

## 13. 前端实现检查清单

每次修改 UI 前后检查：

- 是否复用现有 CSS variables？
- 浅色和深色模式是否都可读？
- 移动端是否保留足够边距？
- 动效是否克制？
- 是否避免了通用 SaaS 视觉？
- 是否没有硬编码不必要的新颜色？
- 是否没有把正文区做成过重卡片？
- 是否保留了 Zephyr Blog 的纸感、暖棕、东方气质？

## 14. 给 Claude Code 的设计指令摘要

当 Claude Code 修改 zephyr-blog 前端时，优先遵守：

```text
1. Read Style.md first.
2. Reuse src/styles/global.css tokens and .glass-panel.
3. Keep the warm paper / terracotta / serif / glass style.
4. Support dark mode through CSS variables.
5. Do not introduce a separate visual system.
6. Keep animations subtle and readable.
7. Do not output or commit secrets.
```
